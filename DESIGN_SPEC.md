## 視覺風格：2.5D 等距

### 為什麼 2.5D？

- 純 3D（Three.js）→ GPU 需求高、iframe 沙盒限制 → ❌
- 純 2D 平面 → 沒空間感、不可愛 → ❌
- **2.5D 等距 → 純 SVG/CSS、零 GPU、有立體感** → ✅

### 等距參數

- 投影角度：30° 等距（`skewY(26.57°)` / `skewY(-26.57°)`）
- ViewBox：680 × 520（可依內容調整高度）
- 安全區域：x=40 到 x=640
- 桌子等距高度：20px
- 角色站立高度：桌面上方 30-40px

## 角色 Pixel Art 精靈系統

角色使用 **16×20 像素格** 繪製（每像素 = 2×2 SVG units → 32×40 total），風格類似 RPG Maker / Stardew Valley。

### 精靈結構

```
<g transform="translate(x, y)">   // RAF 動畫控制位置（非 CSS transition）
  ├── <ellipse> 影子
  ├── <g> 像素精靈                  // 16×20 grid of <rect> elements
  │   ├── rows 0-3: 頭髮
  │   ├── rows 4-9: 臉（眼、嘴、腮紅）
  │   ├── rows 10-14: 身體（衣服）
  │   └── rows 15-17: 腿 / 鞋子
  ├── <g> 行為指示器圖標            // 根據當前行為顯示小圖標
  ├── <g> 名牌 + 狀態圖標
  └── <BehaviorBubble> 對話氣泡
```

## 行為引擎

### 核心機制

```javascript
// 每個角色獨立的行為計時器（doSchedule setTimeout 鏈）
// 角色 70-80% 時間待在桌前（類似 Stardew Valley NPC 風格）
// 每次行為持續 12-35 秒（工作行為）或 5-18 秒（日常行為）
// 行為切換後如需移動，使用 RAF 動畫平滑走到目的地

function doSchedule(agentId) {
  // try/catch 包裹，確保 setTimeout 鏈永遠不會斷
  // 群體事件進行中時跳過（officeLife 控制行為）
  // 走路中時等待 1.5s 重試（含卡住偵測，15 次後強制重置）
  const weights = getWeights(agent.status);  // 根據狀態選權重
  const category = weightedRandom(weights);   // work/daily/social/away
  const behavior = pickBehavior(category);    // 隨機選行為
  playBehavior(agent, behavior);              // 播放動畫 + 行為指示器
  scheduleNext(agent, behavior.duration);     // 12-35 秒後下一個
}
// 看門狗：每 10 秒檢查行為是否卡住 >45 秒，自動重啟排程
```

## Level B 權重覆蓋

### 覆蓋規則

```javascript
// 基礎權重（所有人預設）— 70-80% 桌前時間
const baseWeights = { work: 65, daily: 12, social: 13, away: 10 };

// Level B 推斷到特定狀態時的覆蓋
const overrides = {
  working: { work: 82, daily: 8, social: 5, away: 5 },
  idle:    { work: 55, daily: 18, social: 15, away: 12 },
  done:    { work: 25, daily: 25, social: 30, away: 20 },
  blocked: { work: 10, daily: 10, social: 10, away: 10, frustrated: 60 },
};
```

### 推斷腳本（inferStatus.sh）

```bash
#!/bin/bash
# 讀取 AgentCortex 的 current_state.md，推斷辦公室狀態
# 完全唯讀，不寫入任何東西

STATE="docs/context/current_state.md"

if [ ! -f "$STATE" ]; then
  echo '{"mode":"demo"}'
  exit 0
fi

PHASE=$(grep -oP '(?i)current\s*phase[:\s]*\K\w+' "$STATE" 2>/dev/null)
ACTIVE=$(grep -oP '(?i)active\s*workflow[s]?[:\s]*\K[/\w]+' "$STATE" 2>/dev/null)

# command → agent 映射
case "$ACTIVE" in
  /bootstrap|/plan|/spec) AGENT="pm" ;;
  /brainstorm|/decide)    AGENT="arch" ;;
  /implement)             AGENT="dev" ;;
  /test|/review|/test-classify) AGENT="qa" ;;
  /ship|/handoff|/retro)  AGENT="ops" ;;
  /research)              AGENT="res" ;;
  *)                      AGENT="none" ;;
esac

echo "{\"mode\":\"agentcortex\",\"phase\":\"$PHASE\",\"active\":\"$AGENT\",\"command\":\"$ACTIVE\"}"
```
